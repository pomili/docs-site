ESP32-S3 网页配网（Web Provisioning）

基于 ESP32-S3 + ESP-IDF v5.3 实现的 Web 配网方案
项目路径：F:\espidf_openclaw\web_prov\web_prov

一、功能概述​

设备启动后自动判断：

  1. 有已保存的 WiFi 凭证 → 尝试连接 STA
  2. 连接成功 → 正常运行
  3. 无凭证 / 连接失败 → 启动 AP 模式 + Web 配网门户

配网门户特点：

  • 📡 自动扫描附近 WiFi 信号
  • 🎯 点击选择网络，只需填密码
  • 🔄 提交后自动保存并重启
  • 🚪 Captive Portal（DNS 劫持）

二、整体架构​

┌─────────────────────────────────────┐
│          app_main()                 │
├─────────────────────────────────────┤
│ 1. NVS 初始化                       │
│ 2. 读取已保存的 WiFi 凭证           │
│ 3. 有凭证 → 尝试 STA 连接           │
│ 4. 无凭证/失败 → AP + Web 门户     │
└─────────────────────────────────────┘
                    │
         ┌─────────┴──────────┐
         ▼                    ▼
   ┌──────────┐        ┌──────────────┐
   │ STA 模式 │        │   AP 模式    │
   │ 正常使用 │        │ Web 服务器   │
   └──────────┘        │ DNS 服务器   │
                       │ WiFi 扫描    │
                       │ NVS 保存     │
                       └──────────────┘

三、核心代码说明​

1. NVS 存储（load_creds_from_nvs / save_creds_to_nvs）​

使用 NVS 命名空间 wifi_config，存储两个键：

键值说明
ssid字符串WiFi 名称
password字符串WiFi 密码

流程：

nvs_open("wifi_config", NVS_READWRITE, &nvs);
nvs_set_str(nvs, "ssid", ssid);
nvs_set_str(nvs, "password", password);
nvs_commit(nvs);
nvs_close(nvs);

2. WiFi 扫描（/api/scan 端点）​

在 AP 模式下切换到 WIFI_MODE_APSTA 执行扫描，返回 JSON：

{
  "networks": [
    {"ssid": "MyWiFi", "rssi": -45, "auth": "WPA2", "bar": 5},
    {"ssid": "Guest", "rssi": -70, "auth": "OPEN", "bar": 3}
  ]
}

关键代码：

ESP_ERROR_CHECK(esp_wifi_set_mode(WIFI_MODE_APSTA));
wifi_scan_config_t scan_cfg = {
    .show_hidden = false,
    .scan_type = WIFI_SCAN_TYPE_ACTIVE,
};
ESP_ERROR_CHECK(esp_wifi_scan_start(&scan_cfg, false));
xSemaphoreTake(s_scan_done, pdMS_TO_TICKS(10000));
// 获取结果并组装 JSON
ESP_ERROR_CHECK(esp_wifi_set_mode(WIFI_MODE_AP)); // 切回纯 AP

3. Web 页面（/ 根路径）​

前端 HTML + 嵌入式 JavaScript，功能：

  • 页面加载时自动调用 /api/scan
  • 将结果渲染为可点击的列表
  • 点击选择 SSID，自动填入隐藏表单
  • 用户只需输入密码

4. 表单提交（/save POST）​

解析 application/x-www-form-urlencoded 格式，URL 解码后保存到 NVS，重启：

save_creds_to_nvs(ssid, password);
vTaskDelay(pdMS_TO_TICKS(1000));
esp_restart();

5. DNS 服务器（Captive Portal）​

监听 UDP 53 端口，将所有 DNS 查询指向 192.168.4.1：

buf[2] |= 0x80;  // Response
buf[qlen++] = 192; buf[qlen++] = 168;
buf[qlen++] = 4; buf[qlen++] = 1;   // 192.168.4.1
sendto(sock, buf, qlen, 0, ...);

五、使用方式​

  1. 搜索 WiFi 热点 ESP32-Config（开放网络）
  2. 连接后自动弹出配网页，或手动访问 http://192.168.4.1
  3. 页面自动扫描附近 WiFi，点击选择
  4. 输入密码，点击「连接网络」
  5. 设备保存配置并重启，自动连接目标 WiFi

七、关键编译配置（sdkconfig）​

配置项值说明
CONFIG_ESP_WIFI_SOFTAPy启用 SoftAP
CONFIG_ESP_HTTP_SERVERy启用 HTTP 服务器
CONFIG_ESP_NVS_ENCRYPTION(默认)NVS 加密
CONFIG_LWIP_DNSy启用 DNS

八、常见问题​

Q: 手机连上 ESP32-Config 但没有弹出配网页？
A: 手动打开浏览器访问 http://192.168.4.1

Q: 扫描不到 WiFi 列表？
A: 点击「重新扫描」按钮重试；确保目标 WiFi 在 2.4GHz 频段

Q: 保存后设备连不上 WiFi？
A: 会清除凭证重新进入配网模式。检查密码是否正确、WiFi 是否为 2.4GHz

Q: ESP32-S3 串口连不上？
A: 检查设备管理器中的串口号，ESP32-S3 内置 USB 串口通常显示为 USB 串行设备(VID_303A)
